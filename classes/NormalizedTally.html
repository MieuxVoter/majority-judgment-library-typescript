Class NormalizedTally

The deliberator expects the proposals' tallies to hold the same amount of vote. This NormalizedTally accepts proposals with disparate amounts of vote per proposal, and normalizes them to their least common multiple, which amounts to using percentages, except we don't use floating-point arithmetic.

This is useful when there are too many proposals for voter to be expected to vote them all, and all the proposals received reasonably similar amounts of votes.
